# Code Reviewer

An AI-powered GitHub code review bot. When a pull request is opened or updated, it clones the branch into an isolated Cloudflare Sandbox, retrieves relevant coding rules from a Supabase vector store, and posts a structured review comment generated by Gemma.

## How it works

```
GitHub PR event
      │
      ▼
Cloudflare Worker  (/webhook)
      │  verify HMAC signature
      │  clone PR branch → Cloudflare Sandbox
      │  read up to 5 changed files (full content + diff)
      │
      ▼
Supabase pgvector
      │  embed patch → similarity search → matched rules
      │
      ▼
Gemma (gemma-4-31b-it via Google AI)
      │  prompt = rules + diff + full file content
      │  traced via LangSmith
      │
      ▼
GitHub PR comment
```

## Stack

| Layer | Technology |
|---|---|
| Rules UI | Next.js 16 (App Router) + Tailwind CSS v4 |
| Vector store | Supabase pgvector |
| Embeddings | `gemini-embedding-exp-03-07` (768-dim) |
| LLM | `gemma-4-31b-it` via Google AI |
| LLM orchestration | LangChain (`@langchain/google-genai`) |
| Observability | LangSmith |
| Webhook runtime | Cloudflare Workers + Cloudflare Sandbox |
| Worker deploy | GitHub Actions (on push to `master` touching `worker/`) |

## Setup

### 1. Supabase

Run `supabase-setup.sql` in your [Supabase SQL editor](https://supabase.com/dashboard) to create the `rules` table, vector index, and `match_rules` RPC function.

### 2. Next.js (Rules UI)

```bash
bun install
```

Create `.env.local`:

```env
NEXT_PUBLIC_SUPABASE_URL=
NEXT_PUBLIC_SUPABASE_ANON_KEY=
SUPABASE_SERVICE_ROLE_KEY=
GOOGLE_API_KEY=
```

```bash
bun dev   # http://localhost:3000
```

### 3. Cloudflare Worker

```bash
cd worker && bun install
```

Set secrets:

```bash
wrangler secret put GOOGLE_API_KEY
wrangler secret put SUPABASE_URL
wrangler secret put SUPABASE_SERVICE_ROLE_KEY
wrangler secret put GITHUB_TOKEN
wrangler secret put GITHUB_WEBHOOK_SECRET
wrangler secret put LANGSMITH_API_KEY
```

Deploy:

```bash
bun run deploy
```

### 4. GitHub Webhook

In your target repo → **Settings → Webhooks → Add webhook**:

- **Payload URL:** `https://code-reviewer-worker.<account>.workers.dev/webhook`
- **Content type:** `application/json`
- **Secret:** same value as `GITHUB_WEBHOOK_SECRET`
- **Events:** select **Pull requests** only

## Using in another repository

Create `.github/workflows/code-review.yml` in the other repo:

```yaml
name: AI Code Review

on:
  pull_request:
    types: [opened, synchronize, reopened]

jobs:
  review:
    uses: YOUR_GITHUB_USERNAME/code-reviewer/.github/workflows/code-review.yml@main
    secrets:
      GOOGLE_API_KEY: ${{ secrets.GOOGLE_API_KEY }}
      SUPABASE_URL: ${{ secrets.SUPABASE_URL }}
      SUPABASE_SERVICE_ROLE_KEY: ${{ secrets.SUPABASE_SERVICE_ROLE_KEY }}
      LANGSMITH_API_KEY: ${{ secrets.LANGSMITH_API_KEY }}
```

This repo must be **public** for cross-repo reuse.
